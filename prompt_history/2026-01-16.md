# Session 2026-01-16

## Session 1 - Handler USB Power (uhubctl)

### Contexte
Suite de la session précédente (perdue à cause d'une déconnexion) sur le pilotage USB via `uhubctl` pour zap-to-dispenser.

### Objectif
Créer un handler `usb_power` pour contrôler l'alimentation des ports USB en alternative au servo GPIO.

### Outil utilisé
`uhubctl` - contrôle per-port power switching sur hubs USB compatibles.

### Commandes de base
```bash
# Lister les hubs disponibles
sudo uhubctl

# Allumer/éteindre un port (nécessite -l pour spécifier le hub)
sudo uhubctl -l <location> -a on -p <port>
sudo uhubctl -l <location> -a off -p <port>
```

### Résultat uhubctl
```
Hub 2 : USB 3.0 intégré (4 ports PPPS)
Hub 1-1 : Hub USB 2.0 externe (4 ports PPPS) ← Recommandé pour distributeur
Hub 1 : Contrôleur xHCI (1 port, hub 1-1 branché dessus)
```

### Implémentation finale
- Handler `usb_power` complet avec actions: on, off, toggle, cycle, status
- Hub/port dynamiques via variables workflow
- Config par défaut: `default_hub: "1-1"`
- Workflows exemples: `zap-to-usb-dispenser.yml.example`, `zap-to-usb-dispenser-error.yml.example`
