## Session 2 - Investigation zaps non détectés

### Problème
Les zaps envoyés vers un npub configuré dans `zapRecipients` n'étaient pas détectés par PipeliNostr, alors qu'ils étaient visibles sur Iris.to.

### Diagnostic
Le filtre whitelist précoce (ligne 242-245 de `nostr-listener.ts`) vérifiait `event.pubkey` pour tous les événements sauf Gift Wrap (kind 1059).

**Problème** : Pour les zap receipts (kind 9735), `event.pubkey` est la pubkey du **service LNURL** (Wallet of Satoshi, Alby, etc.), pas celle du sender du zap. Le vrai sender est dans le tag `description`.

Donc si whitelist activée → zaps silencieusement ignorés car la pubkey de WoS n'est pas dans la whitelist.

### Fix
Exclusion des zaps (kind 9735) du filtre whitelist précoce, comme pour Gift Wrap.

### Commits
- `5b32dc2 fix: exclude zap receipts (kind 9735) from early whitelist check`
- `63231a5 fix: use real zap sender for whitelist/from_npubs filters`

### Détail technique
Pour les zaps (kind 9735), `event.pubkey` est la pubkey du LNURL provider (WoS, Alby, etc.), pas celle du sender. Le vrai sender est dans le tag `description` (zap request JSON), parsé par `zap-parser.ts`.

Deux corrections nécessaires :
1. `nostr-listener.ts` : exclure kind 9735 du filtre whitelist précoce (comme kind 1059)
2. `workflow-matcher.ts` : utiliser `parsedZap.sender.pubkey` au lieu de `event.pubkey` pour les filtres `from_whitelist` et `from_npubs`

## Session 3 - Zap to GPIO Dispenser

### Demande
Créer un workflow pour distribuer du vin via GPIO quand un zap est reçu, avec possibilité de multi-bouteilles.

### Implémentation
1. **Workflow `zap-to-gpio-dispenser`** : pulse GPIO via transistor NPN
2. **Fix workflow** : `to` au lieu de `recipient` pour nostr_dm, `zap_min_amount` en valeur directe
3. **Nouveau filtre `zap_event_id`** : filtrer les zaps sur une note spécifique (pour multi-bouteilles)

### Câblage
- GPIO 3.3V → Résistance 330Ω → Base transistor NPN (2N2222)
- +5V → M+ moteur
- M- moteur → Collecteur transistor
- Émetteur transistor → GND
- Note : GPIO BCM 17 = pin physique 11

### Commits
- `4d606f2 feat: add zap-to-gpio-dispenser workflow`
- `c7bbb9c fix: zap-to-gpio-dispenser workflow`
- `eadf60d feat: add zap_event_id filter for note-specific zaps`

### Résultat
Distributeur multi-bouteilles fonctionnel : chaque note Nostr = une bouteille différente (vin rouge, vin blanc, etc.)
