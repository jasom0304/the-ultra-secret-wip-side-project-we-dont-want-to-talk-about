- L'utilisateur a un buzzer **passif** (pas actif)
- Buzzer passif = nécessite PWM pour générer un son
- Tests manuels via `pigs` confirmés :
  - Software PWM : `pigs pfs 27 800` + `pigs p 27 128` (fonctionne)
  - Hardware PWM : `pigs hp 18 800 500000` (fonctionne sur GPIO 18)

- `tone_freq: 0` = buzzer actif, simple on/off

Utilise `pwmFrequency()` et `pwmWrite()` de pigpio-client pour générer le ton.

## Session 2 - Affichage LCD I2C

### Demande
Intégrer un écran LCD 20x4 I2C (FREENOVE B0B76VGXYH) pour afficher le statut des workflows :
- Ligne 1 : "Processing..." / "Done!" / "Failed!"
- Ligne 2 : Nom du workflow
- Ligne 3 : Source du trigger (nom npub ou "HTTP"/"Hook"/"Manual")
- Ligne 4 : "Wait for it!"
- En idle : "PipeliNostr" et "Waiting to be awesome!"

### Fichiers créés/modifiés
- `src/core/lcd-display.ts` (nouveau) - Gestionnaire LCD via i2cset
- `src/config/schema.ts` - Schéma config LCD
- `src/core/workflow-engine.ts` - Appels LCD sur start/complete
- `src/index.ts` - Init/shutdown LCD
- `config/config.yml.example` - Section config LCD

### Décisions techniques
- Utilise `i2cset` directement (pas de dépendance npm pour I2C)
- Communication 4-bit avec PCF8574 backpack
- Mapping npub->nom optionnel dans config

### Commits
- `f541956` feat: add LCD display support for workflow status

### Configuration serveur requise
```yaml
# Dans config/config.yml
lcd:
  enabled: true
  i2c_bus: 1
  i2c_address: 0x27  # ou 0x3F selon module
  # npub_names:
  #   "npub1...": "Alice"
```

### Prérequis Raspberry Pi
```bash
sudo apt install i2c-tools
sudo raspi-config  # Interface Options -> I2C -> Enable
```

### Amélioration : Récupération dynamique des noms de profil

Demande : au lieu d'utiliser une config statique `npub_names`, récupérer le nom depuis les relays Nostr.

#### Implémentation
- Ajout cache de profils dans `lcd-display.ts`
- Fetch kind 0 (metadata) depuis les relays configurés
- Utilise SimplePool de nostr-tools avec timeout 3s
- Pre-fetch en background pendant l'exécution du workflow
- Mise à jour de l'affichage quand le nom est récupéré

#### Fichiers modifiés
- `src/core/lcd-display.ts` : méthodes `setRelayUrls()`, `fetchProfileName()`, `doFetchProfile()`, `prefetchProfile()`
- `src/core/workflow-engine.ts` : appel `prefetchProfile()` en background
- `src/index.ts` : appel `lcdDisplay.setRelayUrls(config.relays.primary)`
